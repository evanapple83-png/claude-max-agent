# Brutalist / technical direction

For data-heavy, utilitarian, or deliberately raw interfaces where information density and honesty matter.

- **Grid is king.** A rigid, visible structure — strong columns, hairline rules, monospace or grotesque type. Let the grid show.
- **Type:** extreme scale contrast (tiny labels against large numbers/headings), often a mono or neo-grotesque face. Uppercase labels, tabular figures for data.
- **Color:** utilitarian and sparse — mostly neutral, with functional accents (status colors, one highlight). Color carries *meaning* (state, category), not decoration.
- **Density:** embrace it where the user wants data — tight tables, compact rows, information per pixel. But keep alignment ruthless so density doesn't become noise.
- **Surfaces:** flat, hard edges, square corners, borders over shadows. Texture/analog degradation (grain, scanlines) only if it serves the concept, never as filler.
- **Motion:** minimal and mechanical — instant state changes, snappy transitions. No decorative flourish; this direction values directness.
- **Honesty:** show the real data, real states (loading, empty, error), real precision. This direction looks bad when it hides complexity behind polish.
